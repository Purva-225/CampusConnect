# Contributing to CampusConnect 🎓

Thank you for contributing to CampusConnect! This guide will help you get started.

## 🚀 Getting Started

1. **Fork** the repository
2. **Clone** your fork: `git clone https://github.com/YOUR_USERNAME/CampusConnect.git`
3. Follow the setup in [README.md](README.md)
4. Create a branch: `git checkout -b feat/your-feature-name`

## 📋 How to Contribute

### Finding Issues
- Look for `good-first-issue` label for beginner-friendly tasks
- Look for `help-wanted` for tasks needing contributors
- Comment "I'd like to work on this" before starting

### Making Changes
- Keep commits small and focused
- Use conventional commit messages:
  - `feat: add new feature`
  - `fix: fix a bug`
  - `docs: update documentation`
  - `style: formatting changes`

### Submitting PR
1. Push to your fork
2. Open a Pull Request against `master` branch
3. Fill in the PR template
4. Wait for review (usually within 48 hours)

## 🏗️ Code Style

- Use **inline styles** for React components (project standard)
- Keep components in `client/src/pages/` or `client/src/components/`
- Backend routes go in `server/routes/`
- Always handle errors with try/catch

## 🌿 Branch Naming

- Features: `feat/feature-name`
- Bug fixes: `fix/bug-description`
- Documentation: `docs/what-you-updated`

## ❓ Need Help?

Open a discussion or comment on the issue. We're friendly! 😊
